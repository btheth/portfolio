Benjamin Heth

By day a QA tester at Conversant, an internet advertising company. I work with technologies like Spark and Hive to test big data applications.

In my free time, I teach myself web development. This profile showcases the results of those efforts.

Portfolio
Finterest
My final freecodecamp project, a fish-themed pinterest clone with a React front end, an Express backend, and MongoDB. At one point in time (for about a day) you could log in with Facebook, before they blocked the site for content violations, and I didn't go through the trouble to fight it since this was just a learning exercise. Nontheless, I am proud of the end result of this project, and overall this is the project I think turned out the best visually.
BookSwap
A website for trading books. Users can add books to their profile, put them up for trade, and then request to trade them for other user's books. The other user can refuse or deny the trade, and the initiator can also cancel the trade. I really enjoyed this project because it felt like the first time I was putting together absolutely everything I had learned up to this point. I utilized Google's book api in the book search feature. This was my first full-stack project utilizing React, so I learned a lot from it.
Dungeon Crawler
I spent a lot of time on this project, and it is definitely the one I had the most fun making. You have to fight your way through 4 levels, finding weapons and health packs, to defeat the boss at the end. All of it is home-grown, including the 'level generation' algorithm, which is far from perfect (it tends to skew up and to the left), and I'm sure there are much better options I could have found on the internet, but I am still pretty proud of it. The project was built with React.
Voting App
This was my first full-stack application. Users can create polls with any number of options, and responders can select an existing or new option. Anyone can view them, but you must be logged in to create a poll. You can also see a graph of the results. It uses plain JQuery (combined with Pug) on the front end, with an Express backend and MongoDB. Since it was my first full-stack project, I learned a lot about how all of the pieces really fit together, as well as other aspects like NPM and deployment.
Simon Game
A javascript recreation of a classic game. Even though this was made with JQuery, I had to dabble in 'state management' for the first time in ways that would help me better understand React when I started learning it later. It was also my first introduction to asynchronous concepts as the whole game leans heavily on the setTimeout function.
Stock Market Chart
Not my first React app, and not my first D3 app, but this was the union of those two libraries. It's not all that pretty but I enjoyed putting it together. Users can add or remove stocks, and the results will affect any other user currently looking at the page as well. It was my introduction to web sockets, as they were necessary for those core features. I also had to implement a sort of primitive, home-grown queue system for adding stocks because the API I use for stock information can be very slow at times, which was a fun challenge.
Recipe Book
This was the third React application I put together, but I felt it was the first one with real substance to it. The user can add, remove, and edit their recipes via the modal popup, and collapse them at the recipe, ingredient, or instruction level on the main page. Making all of this work together with React was a very fun challenge. This was also my first introduction to local storage - your recipes will be saved and show up the same way next time you visit the page.
Random Quote Machine
A simple random quote generator. The user can click to see a random quote, and click another button to Tweet the quote. I included it in my portfolio because even though it seems simple looking back, it was my very first introduction to using an API to fetch data, and then dynamically manipulating that data in to my application.
Javascript Calculator
A calculator made with javscript. It has all the functionality you would expect out of a calculator, with some artificial limits on number length for the sake of a learning exercise. An introduction to handling potentially unexpected user input and thinking through all of the different conditions that may present themselves.
coordiNight
An application driven by the Yelp Fusion API that lets a user search for a location and see a list of night life spots around it. The user can see how many people are going, or interested in going, to the location. They can also click either of these buttons themselves to indicate if they would like to go. Unfortunately I recently had to limit the results per search to 5 due to API changes.
Game of Life
The pre-cursor to the Dungeon Crawler, at least in my mind, included in my portfolio because it is fun to look at. Another React project based on Conway's Game of Life, this was a lesson in managing a complex, always changing state, with several different settings.
Global Temp Heatmap
A heatmap built with D3 showing deviation from average global temperature by month, where earch rectangle is a different color based on it's deviation and shows more information on mousover..
Meteor Landing Map
A map of the globe that shows the landing point of all meteors that have hit Earth. Each dot is larger or smaller based on the impact size and shows more information on mousover.
Country Border Force Graph
A force directed graph with a dot representing each country whose size is determined by how many countries it borders. Each dot has the countries flag in the middle, and pushes away other dots from it to create the force effect.
GDP Barchart
A bar graph made with D3 that shows GDP per year of the selected country. The user can select any country from the drop down list to display it's GDP.
Bike Doping Scatterplot
A scatterplot graph built with D3 where each dot represents a race by a single biker. The Y-Axis represents "place" and the X-Axis represents seconds behind 1st place. Each dot is red or green based on if the biker has had doping allegations, and also shows information about the biker and race on mousover.
Pomodoro Clock
Based on a theory that being lazy sometimes is OK. The user can pick how much time they want to work, and then how long of a break they want after. Once started, the timer will loop over and over again until paused or reset. The bar at the bottom will also fill up based on how much time has passed.
Tic Tac Toe
A javascript tic tac toe game where the user picks their desired piece, and then faces off against a completely random computer opponent. There's no AI at all behind it. If you lose you are bad at tic tac toe.
Twitch Status Viewer
The user can see which of a list of pre-set Twitch streamers is on or offline, and filter so that they only see one or the other. Another early API education project that utitlized Twitch's own API and gives a small amount of control to the user.
Wikipedia Viewer
Another early project utilizing Wikipedia's API. The user can search for a term, and will get a set of results back showing information about matching terms from their Wikipedia pages.
Local Weather
This app gets the user's geolocation data of the user, and then uses that to determine the weather. It shows a dynamic background image based on the current state of the weather, as well as the temperature in either fahrenheit or celcius.
